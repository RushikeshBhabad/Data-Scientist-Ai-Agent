                                 PROJECT OVERVIEW
================================================================================

This project is an AI-powered Data Science Assistant built with Streamlit. It 
provides an end-to-end solution for data scientists, featuring:
- Dataset Discovery & Search across multiple sources
- Exploratory Data Analysis (EDA) automation
- ML Model Training with AutoML capabilities
- Interactive Chatbot for data science Q&A

The application uses LangChain with Groq's LLM models (LLaMA 3-70B and DeepSeek) 
to provide intelligent, context-aware responses.

FUNCTION: patch_think_blocks()

```python
def patch_think_blocks(code: str) -> str:
    code = re.sub(r'<think>(.*?)</think>', 
                  r'"""\n<think>\1</think>\n"""', 
                  code, flags=re.DOTALL)
    code = re.sub(r"```python\s*", "", code)
    code = re.sub(r"```", "", code)
    return code.strip()
```

PURPOSE: Clean LLM-generated code for execution

PROBLEM SOLVED:
- DeepSeek model uses <think> tags for reasoning
- These aren't valid Python syntax
- Also removes markdown code fences

REGEX BREAKDOWN:
- `<think>(.*?)</think>`: Matches think blocks (non-greedy)
- `flags=re.DOTALL`: Allows . to match newlines
- Wraps content in Python docstrings for validity

def safe_execute(code: str) -> str:
    try:
        local_vars = {}
        exec(code, {}, local_vars)
        return str(local_vars)
    except Exception as e:

def generate_response(user_input, session_id, use_dataframe_agent=False):
    st.session_state.system_status = "processing"
    session = st.session_state.chat_sessions[session_id]
    
    # Build context
    context = "You are a helpful Data Scientist Assistant..."
    
    if session['dataframe'] is not None:
        df_info = f"\nDataFrame: Shape {session['dataframe'].shape}..."
        context += df_info
    
    # Add chat history
    if session['messages']:
        recent_history = session['messages'][-6:]
        history_context = "\n\nRecent conversation:\n"
        for msg in recent_history:
            history_context += f"{msg['role']}: {msg['content'][:200]}...\n"
        context += history_context
    
    # Generate response
    full_prompt = f"{context}\n\nUser: {user_input}\n\nAssistant:"
    message = HumanMessage(content=full_prompt)
    response = llm.invoke([message])
    
    st.session_state.system_status = "online"
    return response.content
```

CONTEXT BUILDING:
1. System prompt (Data Scientist role)
2. DataFrame metadata (if loaded)
3. Recent conversation history (last 6 messages)
4. User's current question

WHY 6 MESSAGES:
- Balances context vs token usage
- ~200 chars each = ~1200 chars history
- Fits comfortably in 8192 token window

STATUS UPDATES:
- Sets "processing" during generation
- Returns to "online" after completion
- Shows "error" on exceptions

def create_dataframe_agent_response(user_input, session_id):

            verbose=True,
            allow_dangerous_code=True
        )
        response = agent.run(user_input)
        return response
```

                         INTERVIEW COMMON QUESTIONS

Q1: "What is the architecture of this application?"

A: Multi-page Streamlit application with:
   - Main entry point (app_main.py)
   - Four feature modules (Dataset, EDA, Model, Chatbot)
   - Page wrappers in pages/ directory
   - Centralized API configuration
   - Session state for persistence

Q2: "How do you handle LLM integration?"

A: Using LangChain with Groq:
   - ChatGroq wrapper for API calls
   - HumanMessage schema for prompts
   - pandas DataFrame Agent for data queries
   - Caching with @st.cache_resource
   - Error handling with graceful degradation

Q3: "Explain the caching strategy"

A: Two types:
   - @st.cache_resource: For non-serializable objects (LLM connections)
   - @st.cache_data: For serializable returns (analysis results)
   - Session state: For user-specific data (uploads, chat history)

Q4: "How does the AutoML pipeline work?"

A: PyCaret-based:
   - Detect task type (classification/regression)
   - Setup preprocessing pipeline
   - Compare multiple models
   - Select best by metric
   - Fallback to sklearn if PyCaret fails
   - Export with save_model()

Q5: "How do you handle errors in LLM responses?"

A: Multi-layer approach:
   - safe_agent_run() wrapper
   - Parse error messages for responses
   - Simplify prompts on failure
   - Return user-friendly error messages
   - Fallback to alternative methods

Q6: "Explain the chat session management"

A: UUID-based sessions:
   - Each session has unique ID
   - Stores messages, DataFrame, files
   - Context built from history
   - Sessions persisted in session_state
   - CRUD operations (create, switch, delete)
